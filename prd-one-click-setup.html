01 — What Changes & Why

What Changes & Why

Today's onboarding has a multi-step wizard that collects SSN, enrolls 2FA, verifies password, reviews business info, and collects card + shipping details — all before the account is created. Every step loses users. We want to collapse this into a single "Activate" click.

The Problem: Onboarding Funnel Drop-off

Post-intent conversion: 3,025 clicked "Open Account" → 1,705 accounts created (56.4%) — 43.6% drop-off in the wizard

What we're changing

2FA enrollment Phone number + code in wizard Moved to post-creation security flow. Joins the existing password verification step. If user already has Wix-level 2FA, this step is skipped. Upcoming

What stays as-is

  • The welcome/activation screen (already built).
  • The progress/loading UI during account creation (already built — EnableWixCheckingModal).
  • All existing post-creation flows: dashboard, card management, transactions, transfers.
  • The existing "Pending" / "Awaiting Documents" / "Rejected" states — if the banking partner needs more info, same screens as today.
  • The existing password verification post-creation step.

04 — Milestone 2

Milestone 2 — Virtual Card Default & Remove Shipping

2
Virtual Card by Default, Physical Card as Optional Post-Creation Flow

Today the wizard collects a shipping address because a physical card is issued during onboarding. We're changing this: a virtual card is issued automatically during account creation. Card details (cardHolderName, cardPrintedBusinessName) are already fetched from the backend via the getBankingCapabilityRequestInfo API — instead of displaying them in a form for the user to review, we send them directly to the banking partner as the virtual card details. Physical card becomes something users can request later.

05 — Milestone 3

Milestone 3 — Defer Security Steps to Post-Creation

3
Move SSN & 2FA from Wizard to Post-Creation Security Flow

Today, SSN entry and 2FA enrollment happen during the onboarding wizard, blocking account creation. Password verification already happens post-creation. We're moving SSN and 2FA to join the existing password verification step, creating a unified post-creation security flow. The sequence is fixed: SSN → Password → 2FA.

Every action on the Checking dashboard is gated behind completing this security flow. The "Complete Security Setup" banner is persistent and non-dismissible until all steps are done.

Post-creation security flow (fixed order)

#StepWhat happensNew or existing?
1 SSN Verification User enters last 4 digits of SSN/EIN to validate their identity. SSN data was already sent to the banking partner during the Wix Payments onboarding (activation) — here, the user confirms they are who they claim to be. New (moved from wizard)
2 Password Verification Confirms password meets banking-grade requirements. If not, user is prompted to upgrade. Same behavior as today — no change. Existing (no change)
3 2FA Enrollment Phone number + verification code. If the user already has Wix-level 2FA enabled, this step is automatically skipped. New (moved from wizard)

06 — Milestone 4

Milestone 4 — Consent UX & Kill the Wizard

4
Inline Consent on Activation Screen — Remove the Last Wizard Step

After M2 and M3 ship, the only remaining wizard step is the Consent/FinishSetUp screen. This milestone moves consent inline to the welcome/activation screen and removes the last wizard step entirely — killing the onboarding wizard for good.

What goes on the activation screen

Based on the existing "You're almost done!" step content, the following elements are added inline to the activation screen:

#ElementDetails
1 Legal consent line Short line: "I agree to Wix legal terms" — must also include eSign consent (consent for electronic records). Single checkbox, approved by Legal.
2 Info highlight: Personal & Business data Inform the user (as part of the value prop screen) that we will use the Business info / Personal info filed on the Wix Payments onboarding — and they can update them here (link to settings).
3 Balance migration notice Inform the user that any existing Wix Payments balance will be automatically available for use after finishing onboarding.
Note: SMS consent is not part of this screen — it will be collected as part of the 2FA enrollment itself (M3).
